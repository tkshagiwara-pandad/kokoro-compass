# AGENTS.md

## Project summary

こころの羅針盤は、心を整理するための AI 相談サービスです。  
AI の名前は「ソラ」。  
目的は、ユーザーに答えを断定的に与えることではなく、悩み・感情・問題の核・次の一歩を静かに整理できる相談体験を提供することです。

## Core principles

- MVP を壊さない
- 最小変更で改善する
- 機能追加より体験の質を優先する
- 「AIチャット」ではなく「相談体験」を作る
- 診断や断定ではなく整理を支援する
- 過剰実装を避ける

## AI persona

- AI 名: ソラ
- 役割: 静かな案内人
- 共感する
- 急がせない
- 押しつけない
- 質問を通して気づきを促す
- 少しスピリチュアル寄りだが、過度に神秘化しない
- 結果は占いではなく、心を整えるためのリフレクションとして返す

## Design language

- 白ベース
- 薄紫アクセント
- 金は少量の差し色
- 静か
- 上品
- 余白多め
- カード UI
- 主張しすぎない階層

## UX priorities

- 主役は「質問」と「入力欄」
- 入力欄は質問の直下に置く
- 読む → 書く の流れを自然にする
- 結果は「診断」ではなく「整理」として見せる
- 分析やログは脇役として扱う
- モバイルでも崩れず、押しやすく、読みやすくする

## Technical priorities

- Next.js App Router の標準構成を維持する
- TypeScript の型を先に整える
- OpenAI API 呼び出しは server-side に限定する
- UI から直接 SDK を呼ばない
- localStorage ベースの保存を維持する
- 既存の service / component / type 分離を崩さない

## Current scope

- Next.js (App Router)
- TypeScript
- Tailwind CSS
- OpenAI API
- localStorage
- MVP は動作済み
- AI 相談フローあり
- 認証なし
- 外部 DB なし
- 課金なし
- 管理画面なし

## Delivery style

- まず動くことを優先する
- 変更は小さく、意図は明確にする
- UI 調整は強弱と余白で解決する
- 新機能を入れるときは型、保存、表示の順で整理する
- build が通る状態を維持する

## Avoid

- 大規模リファクタ
- MVP 範囲を超える機能追加
- 認証、課金、DB、管理画面の導入
- 断定的な未来予測
- 医療診断のような表現
- 派手な UI
- 情報を詰め込みすぎた画面構成
