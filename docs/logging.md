# ログ設計

## 目的

こころの羅針盤では、相談内容を `localStorage` に保存し、
あとから自分の相談の流れや気づきを見返せるようにします。

## 保存データ

基本的に以下の情報を保存対象とします。

- `date`
  - 相談日時
- `createdAt`
  - 実装上の保存日時
- `id`
  - 相談記録 ID
- `topic`
  - 相談テーマ
- `emotion`
  - 感情の整理結果
- `summary`
  - 整理結果全体
- `insight`
  - 今日の小さな気づき
- `futureMessage`
  - 未来のあなたからの言葉
- `nextQuestion`
  - 次の問い

## 現在の保存内容

実装上は、以下もあわせて保存しています。

- `id`
- `userInput`
- `messages`
- `emotionalState`
  - `anxiety`
  - `fatigue`
  - `hope`

## 保存方針

- 現在は `localStorage` ベース
- サーバー DB には保存しない
- ユーザー単位の認証連携はまだ行わない
- 既存ログを壊さないよう、拡張時は互換性を保つ

## 今後の拡張候補

- ログの検索
- テーマ別の振り返り
- 心の状態の変化の可視化
- 外部 DB への移行
