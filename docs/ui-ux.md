# UI / UX 設計

## 相談画面構造

相談画面では、以下の順に情報が流れる構成を基本とします。

1. テーマ選択
2. 共感
3. 質問
4. 入力欄
5. 気づき
6. 未来メッセージ
7. 次の問い
8. 心の状態メーター

## 基本方針

- 主役は「質問」と「入力欄」
- 読む → 書く の流れを自然にする
- 結果は診断ではなく整理として見せる
- ログや分析は脇役にする
- モバイルでも縦の流れが崩れないようにする

## 各要素の役割

### テーマ選択

- 相談の入口を軽くする
- 書き始めやすくする
- AI の整理精度を助ける

### 共感

- 最初に安心感を作る
- ユーザーが話し続けやすくなる土台を作る

### 質問

- 相談体験の中心
- ユーザーの感情や本音を言葉にしやすくする

### 入力欄

- 質問の直下に置く
- 目に入りやすくする
- 会話の主役として強めに見せる

### 気づき

- 会話の中で見えてきた小さな視点を補助的に返す

### 未来メッセージ

- 特別感のあるリフレクションとして見せる
- 占いではなく、やさしい再解釈として扱う

### 次の問い

- 次回利用への導線
- 今回の相談を静かにつなぐ補助要素

### 心の状態メーター

- 補助情報として小さく見せる
- 強い主張はさせない

## デザイン原則

- 白ベース
- 薄紫アクセント
- 金は少量
- 余白を広く取る
- カード UI を基本にする
- 情報の強弱を明確にする
